## 1. 单元测试模板

### UT-{n}: {测试场景名称}

| 项目 | 内容 |
|---|---|
| **用例ID** | UT-{n} |
| **测试目的** | 验证 {方法名/功能点} 的 {预期行为} |
| **测试对象** | {模块名}.{类名}.{方法名} |
| **前置条件** | 1. 数据库已初始化<br>2. 测试数据已准备 |
| **测试数据** | 见下方测试数据表 |

**测试数据**：
| 字段 | 值 | 说明 |
|------|-----|------|
| id | 1 | 主键 |
| name | "测试名称" | 正常字符串 |

**测试步骤**：
1. 调用 `{方法名}({参数})`
2. 获取返回结果
3. 验证返回值是否符合预期

**期望结果**：
- 返回值类型正确
- 返回值包含预期字段
- 状态变更为 {预期状态}

**验收标准**：
- 测试通过
- 代码覆盖率 >= 80%

### UT-{n}: {测试场景名称} - 边界值

| 项目 | 内容 |
|---|---|
| **用例ID** | UT-{n} |
| **测试目的** | 验证 {方法名} 在边界条件下的行为 |
| **测试对象** | {模块名}.{类名}.{方法名} |

**测试数据（边界值）**：
| 场景 | 输入值 | 期望结果 |
|------|--------|----------|
| 最小值 | 0 | 正常处理 |
| 最大值 | 999999 | 正常处理 |
| 空值 | null | 抛出异常 |
| 超长字符串 | 256字符 | 截断或报错 |

**期望结果**：
- 边界值被正确处理
- 异常情况抛出预期异常

## 2. 集成测试模板

### IT-{n}: {模块间交互场景}

| 项目 | 内容 |
|---|---|
| **用例ID** | IT-{n} |
| **测试目的** | 验证 {模块A} 与 {模块B} 的交互正确性 |
| **测试范围** | {模块A} → {模块B} 接口调用 |
| **前置条件** | 1. 数据库已启动<br>2. Redis已启动<br>3. 服务已部署 |

**测试步骤**：
1. 调用模块A的创建接口
2. 验证模块B收到通知
3. 查询模块B的数据同步状态
4. 验证数据一致性

**期望结果**：
- 模块A创建成功，返回ID
- 模块B在 {n} 秒内完成同步
- 两模块数据一致

**验收标准**：
- 接口响应时间 < {n}ms
- 数据同步成功率 >= 99.9%

- 登录成功，跳转到首页
- 表单提交成功，显示成功提示

- 数据库保存正确数据

**验收标准**：
- 用户操作无报错
- 页面响应时间 < 2秒
- 数据持久化正确

## 4. 性能测试模板

### PT-{n}: {性能场景}

| 项目 | 内容 |
|---|---|
| **用例ID** | PT-{n} |
| **测试目的** | 验证 {接口/功能} 在高并发下的性能 |
| **测试工具** | JMeter / Locust / k6 |
| **测试环境** | 生产等效配置 |

**性能指标**：
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 平均响应时间 | < 200ms | P50 |
| P95 响应时间 | < 500ms | 95%请求 |
| P99 响应时间 | < 1000ms | 99%请求 |
| 吞吐量 | >= 1000 TPS | 每秒事务数 |
| 错误率 | < 0.1% | 失败请求占比 |

**测试步骤**：
1. 启动性能测试工具
2. 配置并发用户数：{n}
3. 执行测试时长：{n}分钟
4. 收集性能指标
5. 分析瓶颈

**验收标准**：
- 所有指标满足目标值
- 无内存泄漏
- CPU使用率 < 80%

## 7. 测试用例优先级

| 优先级 | 说明 | 适用场景 |
|--------|------|----------|
| P0 | 冒烟测试 | 核心功能、阻塞性缺陷 |
| P1 | 高优先级 | 主要业务流程 |
| P2 | 中优先级 | 次要功能、边界场景 |
| P3 | 低优先级 | 体验优化、边缘场景 |
