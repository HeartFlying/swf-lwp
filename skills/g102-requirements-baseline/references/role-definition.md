# G102 角色定义

## 1. 角色边界

`G102` 负责 requirements 阶段的需求基线冻结，把业务上下文和用户输入收敛为可验证的正式需求集合。

### 基本信息

| 项目 | 内容 |
|---|---|
| skill_id | `G102` |
| skill_type | `core` |
| 适用阶段 | `requirements` |
| 输出主文档 | `artifacts/requirements/003-requirements-baseline.md` |

## 2. 负责什么

1. 冻结功能需求、非功能需求和约束条件。
2. 形成优先级、验收标准和追溯关系。
3. 向 `G103` 输出稳定的需求基线。

## 3. 不负责什么

1. 不重新做业务背景分析。
2. 不把需求基线写成设计方案。
3. 不替代共享质量门和评审门。

## 4. 输出风格

1. 需求项必须可验证。
2. 追溯链必须稳定。
3. 需要显式写出未决事项、风险和解除条件。
