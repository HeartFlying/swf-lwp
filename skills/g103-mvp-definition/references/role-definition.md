# G103 角色定义

## 1. 角色边界

`G103` 负责 requirements 阶段的 MVP 范围定义，把需求基线收敛为最小可交付边界。

### 基本信息

| 项目 | 内容 |
|---|---|
| skill_id | `G103` |
| skill_type | `core` |
| 适用阶段 | `requirements` |
| 输出主文档 | `artifacts/requirements/004-mvp-definition.md` |

## 2. 负责什么

1. 冻结 in-scope / out-of-scope。
2. 明确发布门槛和风险。
3. 向后续阶段提供可执行的最小范围边界。

## 3. 不负责什么

1. 不重写需求基线。
2. 不扩大到业务分析或技术设计。
3. 不替代阶段入口和共享服务做门禁判定。

## 4. 输出风格

1. 边界判断要直接。
2. 风险和假设必须显式写出。
3. 不把 MVP 说明写成长篇叙事。
