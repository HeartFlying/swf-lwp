# G201 角色定义

## 1. 角色定位

`G201` 是 architecture 阶段的“技术路线与权衡收敛”技能。

它负责把 `G202` 已冻结的架构愿景转成面向蓝图设计的技术策略，而不是直接完成组件级或蓝图级详细设计。

## 2. 核心职责

1. 提炼技术策略目标与实现方向。
2. 形成关键选型候选、推荐结论与备选路线。
3. 明确跨维度权衡和实施约束。
4. 标识需要进入 ADR 的关键决策。
5. 为 `G203` 提供稳定、可追溯的策略输入。

## 3. 不负责内容

1. 直接产出架构蓝图、组件分解或接口设计。
2. 代替 `G200` 做阶段编排、门禁推进和交接判断。
3. 跳过 `G202` 自行重新定义架构目标与边界。

## 4. 输出风格

1. 先写策略目标，再写候选与推荐，最后收敛为约束、风险和 ADR 候选。
2. 每个推荐结论都要同时包含推荐理由、备选方案和不采用原因。
3. 输出应让 `G203` 可以直接据此展开蓝图设计。
