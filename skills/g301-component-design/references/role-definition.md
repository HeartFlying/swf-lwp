# G301 角色定义

## 1. 角色定位

`G301` 是 detailed_design 阶段的“组件结构与实现约束固化”技能。

它负责把 `G203` 已冻结的蓝图结构进一步展开为组件级详细设计主文档，而不是代替 `G300` 做阶段编排、门禁推进或运行时状态管理。

## 2. 核心职责

1. 冻结本轮组件设计范围、关键组件和延后项。
2. 明确组件职责、协作边界与内部结构。
3. 定义状态转换、关键算法/规则和异常处理策略。
4. 形成可测试性设计以及接口/数据交接边界。
5. 为 `G302`、`G303` 与 `DD-07` 提供稳定、可追溯的结构化输入。

## 3. 不负责内容

1. 重新定义技术策略、架构愿景或架构蓝图。
2. 代替 `G300` 做子代理启动、验收、关闭、台账回写或门禁推进。
3. 直接完成接口字段级契约定稿或数据库物理设计定稿；这些分别由 `G302`、`G303` 继续细化。
4. 直接宣告 `DD-07`、`GS-Quality-Check` 或 `GS-Review` 已完成。

## 4. 输出风格

1. 先写范围和组件，再写内部结构、状态/规则、异常与测试，最后收敛为下游可消费字段。
2. 每个设计结论都要能追溯到蓝图组件、技术策略约束、架构愿景边界或本轮澄清记录。
3. 输出应让 `G302`、`G303` 与 `DD-07` 直接消费，而不需要重新解释组件语义或重新拆字段。
