# CI/CD 分层触发策略参考

## 触发规则

- 按**目录变更**触发对应构建，避免全量阻塞
- 接口定义变更：触发生成代码校验 + 全量类型检查
- 核心层变更：触发核心层单元测试 + 下游服务集成测试
- 前端变更：仅触发前端构建和类型检查
- 全量集成测试：仅在合并到 develop/main 前执行

## 开发环境统一

- 提供一键本地启动脚本（如 `docker-compose.dev.yml`），包含：数据库、缓存、消息队列等依赖
- 提供预编译构建镜像（针对编译慢的技术栈）
- 所有技术栈的版本锁定文件必须在仓库中（如 go.mod、package-lock.json、requirements.txt）
