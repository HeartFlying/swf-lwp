# 仓库结构参考

## 推荐：单仓库 + 分层目录

```
project/
├── contracts/           # 单一事实来源：接口定义
│   ├── proto/
│   └── openapi/
├── core/               # 核心处理层
│   └── ...
├── services/           # 业务服务层
│   └── ...
├── frontend/           # 前端层
│   └── ...
├── db/                 # 数据库迁移 + 实体代码（强制同目录）
│   ├── migration/
│   └── model/
├── deploy/             # 部署配置
│   ├── compose/
│   └── systemd/
└── scripts/            # 工具脚本（如类型生成、环境启动）
```

## 决策规则

- 团队 < 8 人、模块间接口密集：单仓库
- 团队 > 15 人、有外包隔离需求：核心仓 + 契约独立子模块
- 禁止：多仓库随意拆分（契约漂移风险高）
