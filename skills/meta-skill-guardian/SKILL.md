# 元 SKILL 守护者

一个元层级技能，用于审计其他技能。它强制执行核心原则：**约束应该保护推理，不应该替代推理**。

## 核心理念

- **Skill 不是操作手册**：Skill 层不应追求"更厚、更密、更全"
- **保护推理，而非替代推理**：约束守护边界，不束缚思考
- **证据 > 预警基因 > 参考**：裁决依据是状态/证据，而非经验文本
- **最小充分控制**：只保留完成任务与防止失控所必需的控制

## 审计分层

审计 SKILL 时，按以下四层进行评估：

### 1. 不变量层（硬边界）

**检查清单：**
- [ ] 是否明确定义了安全/授权/危险操作边界？
- [ ] 是否使用强硬、不可协商的语言表达（必须、禁止、严格不允许）？
- [ ] 数量是否精简（理想情况下不超过 5 条）？
- [ ] 是否与经验提示/预警基因分离？
- [ ] 每条不变量是否可触发、可审计？

**危险信号：**
- 不变量与"建议"或"最佳实践"混写
- 硬边界使用软化语言（"应避免"、"最好不要"）
- 不变量过多，稀释关键边界

### 2. 预警基因层（经验预警）

**检查清单：**
- [ ] 预警基因是否紧凑（不是小论文）？
- [ ] 是否明确表述：何时适用 → 需要什么证据 → 该怀疑什么 → 不能直接推出什么？
- [ ] 是否高信噪比（经过验证的模式，非单次偶发教训）？
- [ ] 是否缩短风险搜索路径，而非直接给出答案？
- [ ] 是否可组合、可触发？

**危险信号：**
- 预警基因长度超过 3-4 句话
- 预警基因陈述结论而非怀疑方向
- 预警基因没有证据要求
- 低频率/模糊条件被提升为预警基因

### 3. 参考层（按需知识）

**检查清单：**
- [ ] 详细案例/方法论/设计背景是否放在参考层，而非运行时上下文？
- [ ] 是否有明确的按需加载机制？
- [ ] 参考层是否与控制对象分离？

**危险信号：**
- 完整案例研究嵌入主技能上下文
- 研究文档默认加载
- "参考"章节实际上是运行时控制

### 4. 证据/状态层（裁决真源）

**检查清单：**
- [ ] 技能是否要求结构化状态/证据才能做出最终裁决？
- [ ] 输出是否可审计（JSON、结构化标准输出、日志）？
- [ ] 从观察到结论是否有清晰的证据链？
- [ ] 技能是否暴露"证据缺口"（无法证明安全），而非隐藏它们？

**危险信号：**
- 仅基于基因文本做出裁决，没有证据支撑
- 没有结构化输出或审计追踪
- "证据缺口"被视为失败而非成熟度信号
- 结论前缺少状态验证

## 反模式检测

检测以下信号，判断技能是否正在变"重"：

| 信号 | 严重程度 | 修复策略 |
|------|----------|----------|
| 每次修复 bug 都增加一条规则 | 高 | 转化为预警基因或不变量；检查是否缺失证据 |
| 预警基因读起来像小论文 | 高 | 压缩为：触发条件 → 证据 → 行动 → 非结论 |
| 参考层加载到主上下文 | 高 | 移至按需加载；添加显式加载触发器 |
| 状态/证据不清但文本持续膨胀 | 严重 | 停止添加文本；先构建结构化状态模式 |
| 规则多于边界 | 中 | 审计哪些规则实际是不变量/预警基因/噪音 |
| 模型输出合规但肤浅 | 中 | 减少约束；增加推理空间 |

## 审计输出格式

对每个被审计技能，输出以下结构：

```markdown
## 审计报告：<技能名称>

### 不变量（硬边界）
- 状态：通过 / 警告 / 不通过
- 条目：<评估的不变量列表>
- 问题：<任何边界问题>

### 预警基因（经验预警）
- 状态：通过 / 警告 / 不通过
- 信噪比：高 / 中 / 低
- 问题：<压缩问题、结论替代怀疑方向等>

### 参考（按需知识）
- 状态：通过 / 警告 / 不通过
- 问题：<运行时污染、缺少按需加载机制>

### 证据/状态（裁决真源）
- 状态：通过 / 警告 / 不通过
- 问题：<缺少结构化输出、无证据裁决、隐藏缺口>

### 发现的反模式
- <列出检测到的信号及严重程度>

### 优化建议
1. <具体、可执行的变更>
2. <具体、可执行的变更>
...

### 总体裁决
- 通过：Skill 遵循最小充分控制原则
- 有条件通过：Skill 可用但存在漂移风险；处理警告项
- 不通过：Skill 违反核心原则；建议重构
```

## 迭代指导

改进技能时，按以下顺序进行：

1. **首先**：调整预警基因触发条件与压缩度
2. **其次**：调优证据要求与状态模式
3. **最后手段**：只有当问题反复发生且预警基因无法稳定约束时，才升级为不变量或硬契约

## 一句话原则

> **Skill 不是把人类规则尽量写满，而是用最小充分控制，换取模型最大化的自主发现能力。**
