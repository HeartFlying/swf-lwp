# 软件FMEA增量知识库

本文件是 `software-fmea-review` 的知识库补充入口，用于沉淀企业历史缺陷、行业风险模式、专项控制要求和可复用评审经验。新增内容必须可追溯、可复用、已脱敏。

## 使用规则

- 追加新知识前，先判断它是“通用风险模式”“行业/系统专项规则”“历史事故案例”还是“控制措施样例”。
- 不保存敏感凭据、客户隐私、真实个人信息、生产密钥、未脱敏日志或不可公开的业务数据。
- 若内容只适用于单个项目，在“历史事故/缺陷案例”中记录，不直接提升为通用规则。
- 若内容与基础库 `software-fmea-reference.md` 不一致，在条目中写明适用范围、优先级原因和证据来源。

## 6. 视频接力跟踪系统风险模式（2026-05-21 补充）

### CASE-20260521-视频接力跟踪断点风险

- 背景：HTVT 高速公路隧道视频接力跟踪系统，80米间距摄像头+30-50米重叠区域
- 失效链 FE/FM/FC：
  - FE：车辆在隧道内丢失，态势统计不准确，应急场景无法定位车辆
  - FM：接力跟踪断点，特征匹配失败，盲区预测错误
  - FC：重叠区域参数不当，低照度环境下特征匹配鲁棒性不足，断点恢复状态机不完整
- 逃逸原因：需求阶段未定义断点恢复机制细节；未验证低照度环境识别能力
- 影响范围：核心差异化能力受损，NFR-003跟踪准确率>=95%可能不达标
- 复盘结论：
  1. 多摄像头接力场景需定义断点阈值和恢复机制
  2. 低照度环境需专项POC验证
  3. 重叠区域参数需可配置
- 后续FMEA检查项：
  - 重叠区域参数范围是否定义
  - 断点恢复时间阈值是否定义
  - 低照度环境准确率是否验证
  - 盲区预测算法是否设计
- 推荐控制措施：
  - 预防：设计断点恢复机制，预留图像增强预处理模块，定义可调匹配阈值
  - 探测：跟踪成功率实时监控，断点告警，连续7天现场测试统计
- 来源/证据：HTVT项目FMEA评审（artifacts/reviews/001b-fmea-risk-review.md）

### CASE-20260521-双机协同关联失败风险

- 背景：HTVT系统洞口卡口摄像机（拍车头）与对向监控摄像机（拍车尾）时空关联
- 失效链 FE/FM/FC：
  - FE：跟踪起点错误，全程轨迹关联失败
  - FM：双机协同关联错误或超时
  - FC：时空关联算法边界未定义，乱序到达处理规则缺失，时钟同步精度不足
- 逃逸原因：需求阶段未定义乱序处理规则（PRD-A002）
- 影响范围：起点错误影响全程跟踪链路
- 复盘结论：
  1. 多机协同场景需定义乱序处理规则
  2. 时钟同步精度需明确要求
  3. 关联失败需保留独立数据供人工复核
- 后续FMEA检查项：
  - 乱序到达处理规则是否定义
  - 关联时间阈值是否定义
  - 关联失败数据处理策略是否定义
- 推荐控制措施：
  - 预防：设计时空关联算法容错机制，定义乱序处理规则，预留人工复核入口
  - 探测：双机协同关联准确率监控（阈值>=98%），关联失败告警
- 来源/证据：HTVT项目FMEA评审（artifacts/reviews/001b-fmea-risk-review.md）

### CASE-20260521-边缘计算算力不足风险

- 背景：HTVT系统82~84路视频并发接入+AI实时推理，部署在2台64路边缘主机
- 失效链 FE/FM/FC：
  - FE：实时性下降，延迟超阈值，系统崩溃
  - FM：并发处理不足，推理延迟堆积
  - FC：边缘主机算力评估不足，推理模型效率问题，未预留冗余
- 逃逸原因：方案设计阶段未完成算力验证（R-003）
- 影响范围：NFR-009并发>=100辆延迟<5秒可能不达标，发布阻断风险
- 复盘结论：
  1. 边缘AI场景需方案设计阶段完成算力评估
  2. 需设计并发降级策略
  3. 需预留增加算力的备选方案
- 后续FMEA检查项：
  - 算力评估是否在方案设计阶段完成
  - 并发降级策略是否设计
  - 算力扩充备选方案是否准备
- 推荐控制措施：
  - 预防：方案设计阶段验证算力，设计并发降级策略，优化推理模型效率
  - 探测：并发压力测试，延迟监控（阈值<5秒）
- 来源/证据：HTVT项目FMEA评审（artifacts/reviews/001b-fmea-risk-review.md）
